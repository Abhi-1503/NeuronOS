01 — The Mark

Three signals, converging into one decision.

The mark isn't a decorative node cluster — it's a diagram of what the product actually does. Scattered inputs (email, documents, meetings) meet at a single point, and one clear signal leaves it. That's the whole thesis, drawn as a logo.

CONSTRUCTION — 3 INPUT VECTORS / 1 OUTPUT VECTOR / 8° NODE OFFSET

Three inputs

Gmail, the CRM, the calendar — every disconnected tool a business already runs on. Drawn as dendrites, not straight lines: nothing about a real business's data arrives in clean geometry.

One node

The Context Engine. Slightly off-center from true geometric middle — a deliberate 8° offset so the mark reads as a moment of synthesis, not a static hub.

One signal out

The recommendation. Bolder stroke, gradient into Spark Cyan — the only warm-bright color in the whole mark, reserved for the moment something actionable leaves the system.

02 — In Motion

The mark doesn't just sit there. It reports on the system's actual state.

Same three states the AI Actions queue already uses — Idle, Thinking, Alert — reused here rather than invented fresh, because a brand's motion language should be the product's motion language.

A slow, single pulse at the signal terminal — nothing needs attention right now.
03 — Color & Type

Built to extend the product's existing palette, not replace it.

Every shipped NeuronOS screen already lives in this indigo family. The brand system doesn't introduce a new palette — it names the one already in use, and adds exactly one new color reserved for a single job.
